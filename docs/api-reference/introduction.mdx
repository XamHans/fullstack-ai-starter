# API Reference

The Fullstack AI Starter provides a robust REST API built with Next.js 15 Route Handlers. All endpoints follow standardized patterns for authentication, error handling, and response formatting.

## Base URL

```
https://yourapp.com/api
```

For local development:
```
http://localhost:3000/api
```

## Authentication

Most API endpoints require authentication using session-based auth powered by better-auth.

### Authentication Methods

### Making Authenticated Requests

For browser-based requests, authentication is handled automatically through cookies. For API clients, you'll need to include the session cookie:

```bash
curl -X GET "https://yourapp.com/api/posts" \
  -H "Cookie: better-auth.session_token=your-session-token"
```

## Response Format

All API responses follow a standardized format:

### Success Response
```json
{
  "success": true,
  "data": {
    // Your data here
  },
  "message": "Optional success message"
}
```

### Error Response
```json
{
  "success": false,
  "error": "Human-readable error message",
  "code": "ERROR_CODE"
}
```

## Status Codes

| Code | Description |
|------|-------------|
| `200` | Success |
| `201` | Created |
| `400` | Bad Request |
| `401` | Unauthorized |
| `403` | Forbidden |
| `404` | Not Found |
| `429` | Too Many Requests |
| `500` | Internal Server Error |

## Error Codes

Common error codes you might encounter:

| Code | Description |
|------|-------------|
| `UNAUTHORIZED` | Authentication required |
| `FORBIDDEN` | Insufficient permissions |
| `INVALID_JSON` | Malformed request body |
| `MISSING_FIELDS` | Required fields missing |
| `POST_NOT_FOUND` | Requested post doesn't exist |
| `USER_NOT_FOUND` | User doesn't exist |

## Rate Limiting

API endpoints are rate-limited to ensure fair usage:

- **Public endpoints**: 100 requests per minute per IP
- **Authenticated endpoints**: 1000 requests per minute per user

Rate limit headers are included in responses:

```http
X-RateLimit-Limit: 100
X-RateLimit-Remaining: 95
X-RateLimit-Reset: 1609459200
```

## Pagination

List endpoints support pagination using query parameters:

```
GET /api/posts?limit=20&offset=0
```

### Parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `limit` | integer | 20 | Number of items to return (max 100) |
| `offset` | integer | 0 | Number of items to skip |

### Response

Paginated responses include metadata:

```json
{
  "success": true,
  "data": [...],
  "pagination": {
    "limit": 20,
    "offset": 0,
    "total": 150,
    "hasMore": true
  }
}
```

## Filtering and Search

Many endpoints support filtering and search:

```
GET /api/posts?search=hello&category=tech&published=true
```

## Example Usage

<CodeGroup>
  ```javascript JavaScript
  // Using fetch
  const response = await fetch('/api/posts', {
    method: 'GET',
    credentials: 'include' // Important for session cookies
  });

  const result = await response.json();

  if (result.success) {
    console.log('Posts:', result.data);
  } else {
    console.error('Error:', result.error);
  }
  ```

  ```bash cURL
  # Get all posts
  curl -X GET "https://yourapp.com/api/posts" \
    -H "Cookie: better-auth.session_token=your-session-token"

  # Create a new post
  curl -X POST "https://yourapp.com/api/posts" \
    -H "Content-Type: application/json" \
    -H "Cookie: better-auth.session_token=your-session-token" \
    -d '{
      "title": "My New Post",
      "content": "This is the content of my post",
      "published": true
    }'
  ```

  ```python Python
  import requests

  # Configure session
  session = requests.Session()
  session.cookies.set('better-auth.session_token', 'your-session-token')

  # Get all posts
  response = session.get('https://yourapp.com/api/posts')

  if response.json()['success']:
      posts = response.json()['data']
      print(f"Found {len(posts)} posts")
  ```
</CodeGroup>
